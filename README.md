GVHMR&GMR 项目整合与使用指南
📌 项目简介
本项目整合 GMR（人体运动重定向） 与 GVHMR（视频人体姿态与形状估计） 两大核心模块，实现「视频人体姿态提取 → 人体运动重定向」的完整流程，可用于数字人动画生成、人体动作迁移、视频姿态重建等场景。
- GVHMR：负责输入视频，完成人体姿态、形状、骨架信息提取
- GMR：基于提取的骨架数据，实现精准人体运动重定向，支持 BVH 动作文件处理

⚙️ 环境安装
为保证两个项目依赖兼容、环境统一，推荐使用 conda 虚拟环境统一管理依赖，全程共用一个环境，避免版本冲突。
1. 创建并激活虚拟环境
# 创建虚拟环境（Python 版本建议 3.8+）
conda create -n hmr-unified python=3.8
# 激活环境
conda activate hmr-unified
2. 安装项目依赖
分别进入 GMR、GVHMR 项目根目录，严格按照各自项目原生 README.md 或 environment.yml 安装依赖：
- 优先保证两大项目核心依赖（PyTorch、OpenCV、SMPL 相关库）版本兼容
- 若出现依赖冲突，以项目运行稳定性为准，微调第三方库版本

下载完成后解压资源包，得到两个核心文件夹：
- body_models：人体姿态模型权重文件（GMR 依赖）
- input：视频输入、骨架预处理资源文件（GVHMR 依赖）
📂 项目文件配置
将解压后的资源文件，分别放置到对应项目指定目录，保证路径匹配，否则会出现模型加载失败、数据读取异常问题。
3.1 GMR 项目配置
将解压得到的 body_models 文件夹，移动至 GMR 项目的 assets/ 目录下，最终路径结构：
GMR/
└── assets/
    └── body_models/  # 人体模型资源文件
3.2 GVHMR 项目配置
将解压得到的 input 文件夹，直接放置在 GVHMR 项目根目录下，最终路径结构：
GVHMR/
└── input/  # 视频、骨架预处理资源文件
🚀 完整使用流程
整体运行逻辑：GVHMR 视频姿态提取 → GMR 运动重定向 / BVH 重定向
4.1 GVHMR 视频骨架提取
1. 进入 GVHMR 项目根目录：cd GVHMR
2. 参考项目原生 README.md 文档，配置视频输入路径、推理参数
3. 运行推理脚本，完成视频人体姿态、骨架信息提取，生成可被 GMR 读取的姿态数据文件
4.2 GMR 人体运动重定向
1. 骨架提取完成后，进入 GMR 项目根目录：cd ../GMR
2. 参考项目 args.txt 中的参数示例，配置输入骨架文件、输出路径、重定向参数
3. 执行运行脚本，完成人体运动姿态重定向推理
4.3 BVH 文件重定向
若需对标准 BVH 动作文件进行运动重定向，具体参数配置、运行脚本、适配规则，可参考 GMR 项目下的 README_unified.md 详细说明。
⚠️ 注意事项
- 路径规范：严格按照本文档配置资源文件路径，禁止中文路径、特殊字符路径，避免模型/数据加载失败
- 环境统一：全程使用同一个虚拟环境运行两大模块，切勿切换不同环境导致依赖报错
- 版本兼容：若运行出现库版本报错，优先参考各项目原生 README.md 或 environment.yml 调整依赖版本
- 资源完整性：确保 body_models、input 文件夹文件完整，无缺失、损坏
❓ 常见问题
- 问题1：模型权重加载失败
解决方案：检查 body_models 路径是否放置在 GMR/assets/ 下，确认文件完整无缺失。
      
- 问题2：GVHMR 无法读取视频输入
解决方案：确认 input文件夹放置在 GVHMR 根目录，检查视频格式是否为项目支持格式。
      
- 问题3：依赖版本冲突报错
解决方案：基于统一虚拟环境，对照两个项目的依赖清单，兼容调整库版本，不建议多环境混用。
